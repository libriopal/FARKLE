# ⚙️ EXECUTION FLOW SUMMARY

1. How a patch enters system: Data is piped via CLI (`npx tsx src/cli.ts <PROMPT> --auto "<prompt text>"`), extracting the reference identity and task string.
2. Where it is stored: Baseline generated files are temporarily isolated and hydrated within `/.ff-verify/temp/`. Active system lockfiles prevent parallel collisions on the real source target.
3. How validation runs: Node's `child_process` securely issues `npx vitest run --reporter=json` returning state evaluation parsed by `validation.service.ts`.
4. How repair loop works: If Vitest fails, a max-5 cycle loop fires in stringent order: TS-Morph AST structure fix -> Safe pattern RegEx -> targeted AI diff replacement -> ultimate AI rewrite. Hashes of current states are tracked (Cycle Break) to catch infinite AI traps.
5. How success is handled: Changes permanently write locally. The block-wide TS type check runs strictly once. Real source is replaced. Branch is cut `patch/<id>`, committed, remote-pushed, and a `gh pr create` CLI fires natively.
6. How failure is handled: If the loop traps or reaches 5 iterations, PRs are bypassed. A git branch is pushed for offline evaluation containing a generated comprehensive trace log inside `/.ff-verify/reports/<PROMPT_ID>_escalation.txt`.

# 🚨 MISSING / RISK DETECTION

### Missing Components:
- There is no central configuration validation mechanism confirming that native packages (like Vitest & gh CLI) actually exist within the host machine before executing child paths.
- The GitService commits everything tracking the active stage, assuming the repo tree is natively pristine prior. It does not actively stash previous modifications.
- Explicit creation logic for `history/` and `locks/` tracking matrices natively missing explicit sub-directory implementations.

### Risky Design Choices:
- Injecting the active code into the real source path (`targetPath`) mid-loop before final validation to appease TS module-resolution paths is inherently risky. If the process is hard-killed (e.g. SIGKILL), `.lock` handlers and `fs.writeFileSync(realSrcBackupBytes)` restore calls won't fire via standard `finally` mechanics.
- Parsing standard error from testing suites explicitly assumes accurate and predictable JSON topologies mappings from Vitest failures.
- Shell command execution using `child_process.execSync` can be susceptible to basic command injection if prompt formatting natively carries malformed characters like `$(reboot)`.

### Over-Engineering:
- The cycle hash trapping (Crypto SHA256 loops tracking) might over-constrain the AI if it attempts identical baseline logic structures with differently ordered imports masking as repetitive traps.
- Imposing strict abstract syntax tree structural updates (`ts-morph`) merely targeting generic faults (e.g. `getDefaultExportSymbol()`) might be superfluous token saving natively offsetting high maintenance configurations versus using tight AI diffs.
